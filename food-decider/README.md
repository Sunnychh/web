# 今天吃什么作战室

一个静态网页工具，用“按口味挑”和“转一下定”两种方式帮你选餐厅。内置范围包括成都华阳到世纪城一带的世豪、红唐、远大、港汇中心、银泰城、保利锦江里、融创等商圈，也补充了重庆大学虎溪校区博士生公寓菊园、东门、大学城熙街、龙湖U城、尖顶坡周边数据。

## 使用

直接用浏览器打开 `index.html`。如果想跑核心逻辑测试：

```bash
node --test tests/food-decider.test.js
```

## 数据说明

`app.js` 内置的餐厅列表是可编辑 seed 数据，字段按大众点评式热度、评分、区域、客单价和分类组织。重庆大学城部分按虎溪校区、大学城熙街、龙湖U城、尖顶坡等公开商圈锚点扩充，并纳入用户订单历史里的重庆大学城/富力城/康居西城/恒大未来城商家；同一商家不同命名会合并权重，非餐饮到综团购、空港店、南京/成都等非附近店和外卖券噪音未导入。大众点评公开页面在自动访问时会触发验证，因此这里没有抓取实时榜单；真实出门前建议再确认营业状态、排队和实时评分。图标已改为本地内联 SVG，不依赖外部 CDN。

页面内支持两种增量方式：

- 在“换个地点”输入商圈关键词，候选席会从内置数据和自定义数据里重新筛选。
- 点击“使用当前位置”，浏览器会请求定位权限，并把经纬度匹配到最近的内置商圈；距离 1km 内算理想命中，3km 内才会刷新候选席，超过 3km 会提示暂未覆盖，不会硬塞成都候选。
- 在“自己加一家”表单里添加餐厅，数据会保存在当前浏览器的 `localStorage`。

当前定位功能不上传位置，也不直接联网搜索餐厅。它使用本地商圈坐标表做近邻匹配。如果要接入实时“大众点评/地图 POI 搜索”，建议另建后端代理服务调用合规 API，再把返回数据转成 `app.js` 里的餐厅字段结构。
